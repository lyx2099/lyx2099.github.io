什么是环境变量

环境变量：bash shell 用来存储有关shell会话和工作环境的相关信息

环境变量的分类

全局环境变量

  • 对shell会话和所有生成的子shell都是可见的
  • 系统变量全大写表示，以区别用户自定义的变量
  • 查看全局变量可使用 env 或 printenv 命令；查看指定环境变量的值只能使用 printenv，不能使用 env。如查看 HOME 环境变量：printenv HOME

局部环境变量

  • 只能在定义它们的进程中可见

  • set 命令显示为某个特定进程设置的所有环境变量，包括局部变量、全局变量以及用户定义变量

设置用户定义变量

设置局部用户定义变量

  • 启动 bash shell 或执行一个 shell 脚本，就能创建在这个shell进程内可见的局部变量。可以通过等号给环境变量赋值，值可以是数值或者字符串

  • 如果要给变量赋一个含有空格的字符串值，必须使用单引号来界定字符串的首和尾

    注意：变量名、等号和值之间不能有空格。如果在赋值表达式中加了空格，bash shell 就会把这个值当做一个单独的命令

  • 设置的局部变量只能在当前 shell 内使用，退出当前shell或者进入子shell这个局部变量就不能使用了

设置全局环境变量

  • 创建全局环境变量的方法是先创建一个局部环境变量，然后使用 export 命令把它导到全局环境中

  • 如果在子shell中创建和父shell同名的局部环境变量，并不会影响到父shell中环境变量的值

删除环境变量

unset 命令（注意引用环境变量时不要加 $）

小窍门：如果要用到变量，使用$；如果要操作变量，不使用 $

如果在子shell中删除全局变量，这只对子shell进程有效。该全局环境变量在父shell中仍然可用，这就和在子shell中修改全局环境变量一样，只在子shell中生效

截止目前所讲到的操作，对环境变量的修改只能持续到退出或重启系统，并不是永久的。下面讲该如何永久保持环境变量的修改效果。

定位系统环境变量

在登入 Linux系统启动bash shell 时，默认情况下 bash shell 会在几个文件中查找命令。这些文件叫做启动文件或者环境文件。bash 检查的启动文件取决于你启动 bash shell 的方式。启动 bash shell 有3 种方式：

  • 登录时作为默认登录shell
  • 作为非登录shell的交互式shell
  • 作为运行脚本的非交互式shell

登录shell

登录shell会从5个不同的启动文件里读取命令

  • /etc/profile

    系统上默认 bash shell 的主启动文件，只要登录Linux系统，bash就会执行 /etc/profile 启动文件中的命令

  • $HOME/.bash_profile

  • $HOME/.bashrc

  • $HOME/.bash_login

  • $HOME/.profile

后面的这四个文件都位于用户的 HOME 目录下。每个用户都可以编辑这些文件并添加自己的环境变量。这些环境变量会在每次启动 bash shell 时生效。shell 会按照下列顺序运行第一个被找到的文件，余下的则被忽略。

环境变量持久化

  • 在 /etc/profile.d 目录中创建一个以 .sh 结尾的文件。把所有新的或修改过的全局变量设置放在这个文件中
  • 存储个人用户永久性bash shell 变量的地方是 $HOME/.bashrc 文件
