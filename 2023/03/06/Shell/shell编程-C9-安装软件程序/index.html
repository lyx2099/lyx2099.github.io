包管理基础

PMS：包管理系统（package mannagement system）

要安装的软件包是被存储在服务器（也可以叫远程仓库）上。使用本地 PMS 搜索新的软件包然后进行安装。软件包通常会依赖于其他的包，在安装软件包之前，PMS会检测依赖关系，并将依赖安装好。

常见的 PMS 基础工具

  • dpkg：基于 Debian 的发行版（如 Ubuntu、Linux Mint）使用 dpkg 命令来安装、管理和删除软件包。
  • rpm：基于 Red Hat 的发行版（如 Fedora、open）使用 rpm 来安装、删除软件。

基于 Debian 的系统

前置知识

dpkg 命令是基于 Debian 系 PMS 工具的核心，包含在这个 PMS 中的其他工具有：

aptitude 工具本质上是 apt 工具和 dpkg 工具的前端。dpkg 是软件包管理系统工具，而 aptitude 则是完整的软件包管理系统。

用 aptitude 管理软件包

  • 如果已安装 aptitude，输入 aptitude 会进入全屏模式。然后下滑到 Installed Package 即可查看已安装的软件包。
  • 显示某个特定软件包的信息：aptitude show package_name
  • 列出所有跟某个特定软件包相关的所有文件列表：dpkg -L package_name
  • 查找某个特定文件属于哪个软件包：dpkg --search absoulte_file_name

用 aptitude 安装软件包

使用 aptitude 安装软件包的步骤：

  • 使用 aptitude search package_name 命令查找软件包。查找结果中每个包之前都会有一个 p 或者 i。i 说明这个包已经安装到系统上；p 或 v 说明这个包可以用，但是尚未安装。
  • 使用 aptitude install package_name 进行安装
  • 检查是否安装成功：使用 aptitude 再次查找软件包，看结果中软件包的名字前面是否出现 i

用 aptitude 更新软件包

  • 将所有已安装的包更新到软件仓库中的最新版本（推荐使用）：aptitude safe-upgrade

  • 不保守的软件升级选项（不检查包之间的依赖关系，将所有软件包升级到最新版本）

用 aptitude 卸载软件包

  • 只删除软件包不删除数据和配置文件：sudo aptitude remove package_name
  • 删除软件包和相关的数据和配置文件：sudo aptitude purge package_name
  • 要看软件包是否已删除：aptitude search package_name 结果中软件包的前面，如果看到一个 c 意味着软件已删除，但配置文件尚未从系统中清除；如果是 p 的话，说明配置文件已删除

aptitude 仓库

aptitude 默认的软件仓库存储在文件 /etc/apt/sources.list 中。aptitude 在搜索、下载或更新软件时，只会检查这些库。如果要为 PMS 添加一些额外的软件仓，就修改这个文件

基于 Red Hat 的系统

前置知识

Red Hat 有几种不同的可用的前端工具：

  • yum：在 Red Hat 和 Fedora 中使用

  • urpm：在 Mandriva 中使用

  • zypper：在 openSUSE 中使用

列出已安装包

  • 找出系统上已安装的包：yum list installed 输出的信息可能一闪而过。最好将结果重定向到一个文件中 yum list installed > installed_software ，然后使用 more 或 less 命令进行查看
  • 查看包是否已安装：yum list package_name
  • 找出系统上某个特定文件属于哪个软件包：yum provides file_name

用 yum 安装软件

用 yum 更新软件

  • 列出所有已安装包的可用更新：yum list updates
  • 对更新列表中的所有包进行更新：yum update
  • 更新某个特定的软件包：yum update package_name

用 yum 卸载软件

  • 只删除软件包而保留配置文件和数据文件：yum remove package_name
  • 删除软件和所有文件：yum erase package_name

处理损坏的包依赖关系

在安装多个软件包时，某个包的软件依赖关系可能会被另一个包的覆盖掉。这就叫做损坏的包依赖关系。出现损坏的包依赖关系修复步骤：

  • 先试试 yum clean all
  • 然后使用 yum update
  • 如果还解决不了问题，使用 yum deplist package_name
  • 仍未解决，最后一招：yum update --skip-broken 该命令的--skip-broken 允许你忽略依赖关系损坏的包，继续更新其他软件包

yum 软件仓库

  • 查看从哪些仓库获取软件：yum repolist
  • 如果仓库没有需要的软件，编辑 yum 的配置文件（位于 /etc/yum.repos.d）,添加正确的URL，并获得必要的加密密钥

从源码安装

下面用一个案例带你了解软件的解包与安装过程，以系统检测工具 sysstat 为例：

  • 去官网（http://sebastien.godard.pagesperso-orange.fr/）下载sysstat 的tarball 包，单击 download
  • 将文件下载到 linux 上，进行解包。要解包一个软件的 tarball，用 tar 命令：tar -zxvf sysstat-11.1.1.tar.gz
  • 解包后所有文件都放在了 sysstat 目录下，使用 cd sysstat 进入该目录。使用 ls 列出目录内容。查看 README 或 AAAREADME 文件。该文件包含文件安装所需要的所有操作。
  • 为系统配置 sysstat，执行 sysstat 目录下的 configure 文件，./configure
  • 使用 make 命令构建二进制文件。make 命令会编译源码，链接器回味这个包创建最终的可执行文件
  • 以 root 用户登录，执行命令 make install
