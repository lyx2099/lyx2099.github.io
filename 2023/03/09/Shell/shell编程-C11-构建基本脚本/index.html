使用多个命令

shell 脚本的关键在于输入多个命令并处理多个命令的结果。可以在命令行中输入多个命令，一次性执行完成，命令之间用分号隔开。例如：
date ; who

创建 shell 脚本

  • 在文件的第一行指定要使用的 shell。格式为：#!/bin/bash

  • shell 脚本的注释以井号（#）开头，但是以 #! 开头的行是告诉shell是用哪个shell来运行脚本，不是注释

用户变量遵循如下规则

重定向输入和输出

输出重定向

  • 使用大于号（>）将命令的输出发送到一个文件中。语法：command > outputfile
  • 案例：date > today 在该案例中，重定向操作符创建了 today 文件，并将命令的输出重定向到文件中。如果输出文件已经存在了，重定向操作符会用新的文件覆盖已有文件
  • 如果不想覆盖掉源文件内容，而是想将命令的输出追加到已有文件中，使用双大于号（>>）来追加数据

输入重定向

  • 将文件的内容重定向到命令。输入重定向使用小于符号（<）。语法： command < file

  • 记忆技巧：在命令行上，命令总是在左侧，而重定向符号指向数据流动的方向。小于符号说明数据正在从输入文件流向命令。

  • wc 命令：使用 wc命令可以数据中的文本进行统计。默认情况下，它会输出3个值：

    • 文本的行数
    • 文本的词数
    • 文本的字节数

  • 案例：wc < login.py

  • 输出结果：2 11 60 表示 login.py 中有2行、11个单词以及601个字节

管道

将一个命令的输入作为另一个命令的输出，这就是管道。管道符号使用竖线表示（|），它和反斜线（\）位于同一个键。语法：
command1 | command2

管道串起来的两个命令会同时执行。在第一个命令产生输出的同时，输出会立即送给第二个命令。数据传输不会用到任何中间的文件或缓冲区。

案例：利用管道将 rpm命令的输出送入 sort命令来产生结果。

浮点解决方案

bc的基本用法

bc是bash的内建计算器。bash计算器允许在命令行中输入浮点表达式，然后解释并计算该表达式，最后返回结果。bash计算器能够识别：

  • 数字（整数和浮点数）

  • 变量（简单变量和数组）

  • 注释（以#或C语言汇总/* */ 开始的行）

  • 表达式

  • 编程语句（例如 if-then 语句）

  • 函数

在 shell 命令提示符中输入bc 命令访问 bash 计算器。然后就可以进行整数或者浮点数的运算了。要退出bash计算器，输入 quit即可。

shell中每个命令都使用退出状态码告诉shell它已经运行完毕。退出状态码是一个0-255的整数值，在命令结束运行时由命令传给shell。可以捕获这个值并在脚本中使用。

Linux退出状态码表

状态码 描述
0 命令成功结束
1 一般性未知错误
2 不适合的shell命令
126 命令不可执行
127 没找到命令
128 无效的退出参数
128+X 与Linux信号x相关的严重错误
130 通过Ctrl+C终止的命令
255 正常范围之外的退出状态码
