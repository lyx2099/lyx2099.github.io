C2 HTML中的JS元素

<script>元素

<script> 元素有下列8个属性

属性 含义
async 可选。表示应该立即开始下载脚本，但不能阻止其他页面动作，比如下载资源或等待其他脚本加载。只对外部脚本文件有效
defer 可选。表示脚本可以延迟到文档完全被解析和显示之后再执行。只对外部脚本文件有效。
src 可选。表示包含要执行的代码的外部文件。
charset 可选。使用 src 属性指定的代码字符集。
crossorigin 可选。配置相关请求的CORS（跨源资源共享）设置。
integrity 可选。允许比对接收到的资源和指定的加密签名以验证子资源完整性。如果接收到的资源的签名与这个属性指定的签名不匹配，则页面会报错，脚本不会执行。这个属性可以用于确保内容分发网络（CDN，Content Delivery Network）不会提供恶意内容。
language 废弃。
type 可选。

标签位置

方式一：放在 <head> 标签内，但是有两个缺点：

  • 所有的js下载、解析和解释完才开始渲染页面

  • 页面渲染延迟，影响用户体验

方式二：放在 <body>内容的后面

优点：页面加载更快

推迟执行脚本

使用 defer 属性，可以让脚本延迟到整个页面都解析完毕后再运行。简言之，defer 就是告诉浏览器先下载脚本但延迟执行。

注意：defer 属性只对外部脚本文件有效，忽略行内脚本

异步执行脚本

使用 async 属性，可以让脚本异步加载。意识就是脚本不按从上往下的顺序执行。

给脚本添加 async 属性的目的，不必等待脚本下载和执行完毕后再加载页面，同样也不必等该一步脚本下载和执行完毕后再加载其他脚本。

异步脚本保证会在页面的 load 事件前执行，但可能会在 DOMContentLoaded 之前或之后。

async 和 defer 都只适用于外部脚本。

行内代码和外部文件

行内文件和外部代码两种方式，推荐使用外部文件，理由如下：

  • 可维护性强：所有的js文件都放在一个目录中，便于维护。

  • 缓存：如果两个页面都要用到同一个js文件，而这个js文件已经被浏览器缓存，则该文件只需要下载一次。

<noscript元素>

如果浏览器在下列两种情况，可以使用<noscript>元素

① 浏览器不支持js脚本

② 浏览器对脚本的支持被关闭（禁用js脚本）

总结

  • js 插入到HTML页面中有3种方式：

    • 行内代码
    • 外部文件引入
    • 使用 DOM API 动态添加

  • 要包含外部的js文件，需要设置 src 属性

  • 在不使用 async 和 defer 属性的情况下，<script>元素的解释顺序是从上往下的

  • 对不推迟执行的脚本，浏览器必须解释完谓语<script> 元素中的代码，才能继续渲染页面

  • defer 属性把脚本推迟到文档渲染完毕后才执行。推迟的脚本按从上往下的顺序执行。

  • async 属性表示脚本不需要等待其他脚本，同时也不能阻塞文档渲染，即异步加载。异步脚本不能保证按照它们出现在文档中的顺序执行。

  • 使用 <noscript> 元素，可以指定浏览器不支持脚本时显示内容。如果浏览器支持并启用脚本，则 <noscript> 中的任何内容都不会被渲染。
